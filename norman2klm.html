Interaction Models in HCI

This page explains three core interaction models used in Human–Computer Interaction: Norman’s Action Cycle, GOMS, and the Keystroke-Level Model (KLM).

1. Norman's Seven Stages of Action & The Gulfs

Reference: Dix et al., HCI (3rd Ed), Chapter 3, pp. 125–127

Don Norman's model describes the cycle of interaction between a user and a system, focusing on the psychological gaps that occur during a task.

The Seven Stages

  1. Establishing the Goal: The user's high-level intent (e.g., "I need more light").
  2. Forming the Intention: The specific intent to use the system (e.g., "Switch on the desk lamp").
  3. Specifying the Action: Mapping the intention to a sequence of physical movements (e.g., "Reach for the toggle").
  4. Executing the Action: The physical motor movement (e.g., "Physically tap the switch").
  5. Perceiving the System State: Noticing the change in the interface (e.g., "The light is now on").
  6. Interpreting the System State: Assigning meaning to the perception (e.g., "The light is on, so the switch worked").
  7. Evaluating the Outcome: Comparing the new state with the original goal (e.g., "Is there enough light to read?").

The Two Gulfs

  • Gulf of Execution: The difference between the user's formulation of the actions and the actions allowed by the system. A wide gulf means the user finds it difficult to figure out how to operate the system.
  • Gulf of Evaluation: The distance between the physical presentation of the system state and the user's expectation. A wide gulf means the user cannot easily determine if their action was successful.

2. GOMS (Goals, Operators, Methods, Selection Rules)

Reference: Dix et al., HCI (3rd Ed), Chapter 12, pp. 422–424

GOMS is a performance model used to describe the knowledge and behavior of an expert user performing routine tasks.

The Four Elements

  • Goals: What the user wants to achieve (e.g., GOAL: ICONIZE-WINDOW).
  • Operators: Basic actions the user must perform at the motor or mental level (e.g., PRESS-KEY, MOVE-MOUSE).
  • Methods: Decompositions of a goal into a sequence of subgoals and operators (e.g., the "CLOSE-METHOD" involving a menu vs. the "L7-METHOD" using a function key).
  • Selection Rules: The logic used to choose a method when more than one is available (e.g., "If the application is 'blocks', use the L7-METHOD").

3. KLM (Keystroke-Level Model)

Reference: Dix et al., HCI (3rd Ed), Chapter 12, pp. 437–438

KLM is a simplified version of GOMS (specifically a "lowest-level GOMS") that provides quantitative predictions of the time it takes an expert to execute a unit task.

Operators and Execution-Time Prediction

The total time is calculated as: Texecute = TK + TB + TP + TH + TD + TM + TR. Standard times include:

  • K (Keystroke/Tap): 0.12s (expert) to 0.28s (average).
  • B (Mouse Button/Tap): 0.1s to 0.2s.
  • P (Pointing): 1.1s (average).
  • H (Homing): 0.4s to move hands between device and keyboard.
  • M (Mental Preparation): 1.35s for the user to "pause and think".
  • R (System Response): Measured time the user must wait for the system.

4. Comparative Summary: Differences Between Models

Reference: Dix et al., HCI (3rd Ed), Chapter 9, pp. 321–326 (Evaluation); Chapter 12, pp. 422–438

Comparison of Norman's Action Cycle, GOMS, and KLM
Feature Norman's Action Cycle GOMS KLM
Primary Focus Psychology of use; understanding why users succeed or fail. High-level planning; describing how experts perform tasks. Low-level motor actions; predicting time for specific keystrokes.
User Type General users (focus on exploratory learning and errors). Expert users performing routine tasks. Expert users with no errors.
Error Handling Central: Classifies errors into Slips (execution) and Mistakes (intent). Not modeled; assumes error-free expert performance. Not modeled; assumes perfect execution.
Feedback Loop Crucial: Includes Perceive, Interpret, and Evaluate stages. Absent: Assumes perception is perfect and instant. Absent: Focuses only on the motor/execution phase.
